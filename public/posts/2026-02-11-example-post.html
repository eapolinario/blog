Writing in Org-Mode

Posts are written in org-mode, a powerful markup language native to Emacs. The benefits:

  • Plain text that remains readable in any editor
  • Structural editing with outlining and folding
  • Export flexibility to HTML, PDF, and other formats
  • Literate programming support for code examples

This aligns with the blog’s philosophy: clarity over cleverness, durability over trends.

Example Code Block

Here’s a simple example in Python: 

def threshold_state(system):
    """Detect if a system is in a liminal state."""
    if not system.has_stable_invariants():
        return "liminal"
    return "stable"

Lists and Structure

Things to consider when writing:

  1. Assume technical sophistication in the reader
  2. Name tradeoffs explicitly rather than hiding them
  3. Connect technical specifics to broader judgments
  4. Avoid beginner explanations unless explicitly relevant

Bullet points for desiderata:

  • Clarity
  • Precision
  • Intellectual honesty
  • Long-term coherence

Conclusion
